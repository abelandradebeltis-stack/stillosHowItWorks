Ações rápidas

Dica: use contas com permissões nos formulários e mantenha backup dos arquivos IdCarga.xlsx antes de qualquer operação.
Importação dos formulários — Passo a passo
  1. Acesse o Google Drive com a conta que tem acesso aos formulários.
  2. Abra Formulários e selecione o formulário de identificação correto:
    • Identificação Geral
    • Mackenzie
    • Five Formaturas (Five)
  3. Visualize as respostas em modo planilha (Abrir no Planilhas Google → "Arquivo" → "Fazer download" → "TSV").
  4. Faça download em formato TSV e renomeie o arquivo para DADOS.TSV.
  5. Salve DADOS.TSV na pasta correta:
    • Identificação Geral: h:\\bd\\IdGeral
    • Mackenzie: h:\\bd\\IdGeralMackenzie
    • Five: h:\\bd\\IdFive
  6. Abra a planilha IdCarga.xlsx (cada pasta tem seu IdCarga.xlsx). Essa planilha consome DADOS.TSV.
  7. Por segurança, salve IdCarga.xlsx com outro nome (ex.: carga.xlsx). Antes configure todas as colunas como texto (telefone em especial).
Exportar via SSIS — Criando TAB_FICHAS_DE_IDENTIFICACAO_CARGA
  1. Abra SQL Server Integration Services (SSIS).
  2. Crie um pacote que leia carga.xlsx (origem Excel 2016) e escreva em [TAB_FICHAS_DE_IDENTIFICACAO_CARGA] (destino: Microsoft OLE DB Provider for SQL Server).
  3. Conecte ao servidor 2003SRV usando autenticação SQL (credenciais necessárias).
  4. Marque a opção "Excluir linhas na tabela de destino" antes do carregamento.
  5. Execute o pacote e confirme que os dados foram carregados na tabela temporária.
  6. Atualize as tabelas da Pronet (TBL_*) no banco de dados site conforme necessário.
Execução das consultas por formulário

Execute a view correspondente para extrair apenas os novos formandos. Salve o resultado em uma planilha XLSX — esse arquivo será usado no próximo passo.

Execute: ESTA VIEW EXPORTA UMA CARGA PARA TAB_FICHAS_IDENTIFICACAO.sql. Essa view processa [TAB_FICHAS_DE_IDENTIFICACAO_CARGA] e retorna só os novos formandos.

Execute: ESTA VIEW EXPORTA DE TAB_FICHAS_IDENTIFICAO_MACKENCZIE PARA TAB_FICHAS_IDENTIFICACAO.sql. Atenção ao semestre/ano e selecione o contrato na cláusula WHERE. Salve o resultado em XLSX.

Execute: ESTA VIEW EXPORTA DE TAB_FICHAS_IDENTIFICAO_MACKENCZIE FIVE PARA TAB_FICHAS_IDENTIFICACAO.sql. Atenção ao semestre/ano e cláusula WHERE com o contrato. Salve em XLSX.
Importar para TAB_FICHAS_IDENTIFICACAO
  1. Utilize SSIS novamente para exportar o XLSX gerado no passo anterior para a tabela TAB_FICHAS_IDENTIFICACAO.
  2. Faça essa operação da mesma forma que no passo 8 (origem Excel 2016 → destino OLE DB → marque excluir linhas quando necessário).
  3. Verifique integridade e execute verificações adicionais nas tabelas Pronet se aplicável.
Download e tratamento das fotos
  1. Com os formandos carregados em TAB_FICHAS_IDENTIFICACAO, é possível iniciar o download das fotos.
  2. Execute a consulta baixa fotos sem nome.sql, selecionando os contratos desejados na cláusula WHERE. Salve o resultado em XLSX.
  3. Converta o XLSX para JSON (ex.: https://www.aconvert.com/document/xlsx-to-json/) e salve como fotos.json na pasta do projeto Node.
  4. Pelo console (CMD) acesse a pasta do projeto Node e execute: node . — as fotos começarão a ser baixadas para a subpasta fotos.
  5. Após término, entre na pasta fotos e delete arquivos .HEIC (foram convertidos para .jpg).
  6. Redimensione as fotos (altura 5 cm, proporção mantida) usando FastStone. Salve em d:\\resize (Crystal Reports acessará essa pasta).
  7. Abra o projeto Fichas no Visual Studio 2019, selecione contrato e imprima as fichas em PDF. Salve o PDF no diretório da pessoa responsável pela impressão.
Atalhos e dicas rápidas
  • Faça backups antes de sobrescrever tabelas.
  • Configurar colunas como texto evita perda de zeros em telefones.
  • Se houver problema de login no 2003SRV, confirme autenticação SQL e permissões.
  • Use nomes de arquivo padronizados: DADOS.TSV, carga.xlsx, fotos.json.
